Queue workers execute logical tasks whose resource demand is often conditional on payload, execution context, dependency health, worker state, and hidden system dynamics. Existing observability practices expose raw metrics and traces, but they do not by themselves define which observations matter for resource estimation, worker occupancy, profiling overhead, or admission-control decisions.

This paper introduces Task Behavior Graphs, a domain-driven model for profiling queue worker tasks under uncertain resource demand. The model represents each task class as a structured graph connecting intrinsic task features, execution context, resource demand, worker occupancy, entropy, and operational risk. We distinguish raw metrics from semantic features, separate root causes from symptoms and outcomes, and show how task-class-specific manifests can guide instrumentation, feature normalization, profiling policy selection, and residual analysis.

The proposed framework is intended as a foundation for adaptive task profiling systems that avoid global metric lists, support domain-specific extensions, and choose profiling strategies based on predictability, context sensitivity, worker occupancy cost, and consequences of estimation error.
